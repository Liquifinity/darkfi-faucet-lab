# Support

## Community Support

Use GitHub issues for:

- reproducible bugs;
- documentation gaps;
- setup questions;
- testnet observations;
- proposed improvements.

Do not post:

- private keys;
- seed phrases;
- wallet DBs;
- VHDX files;
- raw tx artifacts tied to live coins;
- admin tokens;
- local credentials.

## Security Issues

Use the process in [SECURITY.md](SECURITY.md).

If a private security channel is not available yet, open a minimal GitHub issue asking maintainers to establish one. Do not include exploit details or secrets in a public issue.
